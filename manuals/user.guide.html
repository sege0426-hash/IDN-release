IDN 2.0 일반 사용자 사용자 매뉴얼

일반 사용자가 홈, 네트워크, 요청, 설정 화면을 처음 접해도 바로 따라갈 수 있도록 실제 사용 순서에 맞춰 정리한 운영 가이드다. 화면 이름 설명에 그치지 않고 먼저 볼 항목, 눌러야 할 버튼, 작업 후 확인할 결과까지 함께 안내한다.

1. 개요

본 문서는 일반 사용자 권한으로 로그인했을 때 상단 메뉴에서 실제 접근 가능한 화면을 기준으로 정리한 매뉴얼 초안이다.

일반 사용자는 홈, 네트워크, 멤버, 요청, 설정 메뉴를 사용할 수 있다.

이 문서는 메뉴를 나열하는 방식보다 실제 사용 흐름을 먼저 이해하도록 구성했다. 처음 접속한 사용자는 로그인 이후 홈에서 오늘 확인할 항목을 보고, 네트워크에서 실제 사용 자원을 확인한 다음, 요청함과 설정으로 이어서 확인하면 된다.

1.1 추천 읽기 순서

로그인: 접속 실패 원인과 로그인 후 진입 위치를 먼저 확인한다.

홈: 오늘 바로 확인할 알림, 참여 네트워크 수, 내 장치 상태를 파악한다.

네트워크: 실제로 사용하는 네트워크와 장치 상태, 초대와 요청 흐름을 이해한다.

요청: 받은 요청과 보낸 요청의 상태를 구분해서 읽는 방법을 익힌다.

설정: 내 연락처와 비밀번호, 계정 삭제 전 주의사항을 확인한다.

1.2 일반 사용자 권한 범위

가능한 작업: 홈 상태 확인, 참여 네트워크 조회, 네트워크 생성, 장치 상태 확인, 멤버 조회, 요청 확인, 프로필 수정, 비밀번호 변경, 계정 삭제.

제한되는 작업: 컨트롤러와 중계서버 직접 관리, 조직과 부서 관리, 사용자 관리, 보안 정책 관리, 감사와 관리자 설정.

2. 로그인

일반 사용자는 로그인 화면에서 계정 ID와 비밀번호를 입력해 시스템에 접속한다.

2.1 로그인

▶메뉴 위치: 로그인 화면

▶기능 설명:

일반 사용자 계정으로 시스템에 접속한다.

▶사용 절차:

1. 로그인 화면을 연다.

2. 아이디와 비밀번호를 입력한다.

3. 로봇 확인 항목을 선택한다.

4. 로그인 버튼을 클릭한다.

5. 인증 성공 시 홈 화면으로 이동한다.

▶입력 항목:

항목 설명 입력 가이드
아이디 사용자 계정 ID 필수 입력
비밀번호 계정 비밀번호 필수 입력

▶처리 결과 / 유의사항:

아이디 오입력 여부를 확인한다.

비밀번호 오입력 여부를 확인한다.

비밀번호 만료 여부를 확인한다.

계정 잠금 여부를 확인한다.

로그인 후 상단 메뉴에서 홈, 네트워크, 멤버, 요청, 설정 메뉴를 사용할 수 있다.

▶스크린샷:

화면명 이미지
로그인 화면

4. 사용자 메뉴

일반 사용자의 실제 일상 흐름은 홈에서 상태를 확인하고, 네트워크에서 작업 대상을 찾고, 요청함에서 승인 상태를 보고, 마지막으로 설정에서 개인 정보를 관리하는 순서가 가장 자연스럽다.

4.1 처음 사용하는 사용자를 위한 빠른 이해

홈: 오늘 처리할 알림과 내 장치 상태를 먼저 확인한다.

네트워크: 실제로 사용하는 네트워크와 연결 장치를 확인한다.

요청: 받은 요청과 보낸 요청의 상태를 구분해서 본다.

설정: 이름, 연락처, 비밀번호 같은 개인 정보를 마지막에 정리한다.

5. 기능별 상세 설명

5.1 홈

▶메뉴 위치: 상단 메뉴 > 홈

▶기능 설명:

요청/알림 요약과 참여 중인 네트워크 목록을 확인한다.

선택한 네트워크에 속한 내 장치 상태를 상세 영역에서 확인할 수 있다.

처음 사용하는 사용자는 홈을 시작 화면으로 보고 오늘 처리할 일과 현재 상태를 먼저 확인하면 된다.

▶사용 절차:

1. 홈 메뉴를 선택한다.

2. 요청/알림 영역을 확인한다.

3. 참여 중인 네트워크 목록에서 항목을 선택한다.

4. 우측 상세 영역에서 내 장치 상태를 확인한다.

5. 필요 시 요청함 또는 설정으로 이동한다.

▶처리 결과 / 유의사항:

홈 알림은 읽지 않은 항목 기준으로 표시된다.

네트워크 상세에는 해당 네트워크에서 본인에게 연결된 장치만 표시된다.

클라이언트 다운로드와 메뉴얼 다운로드 링크를 시작 화면에서 바로 확인할 수 있다.

장치 상태가 오프라인이면 네트워크 화면에서 해당 장치와 네트워크 상태를 다시 확인한다.

새 초대나 승인 알림이 보이면 요청 화면으로 이동해 실제 처리 상태를 이어서 확인한다.

▶스크린샷:

화면명 이미지
홈 화면

▶추가 화면:

화면명 이미지
홈 장치 상태 패널

5.2 네트워크

▶메뉴 위치: 상단 메뉴 > 네트워크

▶기능 설명:

참여 중인 네트워크 목록을 조회한다.

새 네트워크 생성, 네트워크 상세 보기, 수정, 삭제, 엔드포인트 생성과 수정, 멤버 초대를 수행할 수 있다.

소유자 여부에 따라 편집 가능 범위가 달라질 수 있다.

네트워크 화면은 실제 사용 자원을 확인하는 핵심 화면으로, 이름보다 상태와 연결 구조를 먼저 읽는 것이 중요하다.

▶사용 절차:

1. 네트워크 메뉴를 선택한다.

2. 검색과 필터로 네트워크를 찾는다.

3. 네트워크 카드를 선택해 상세 화면을 연다.

4. 필요 시 새 네트워크 생성 또는 엔드포인트 생성 작업을 수행한다.

5. 상세 화면에서 상태, 메모, 사용자 할당 관련 작업을 수행한다.

▶화면에서 먼저 이해할 항목:

항목 설명
네트워크 이름 참여 중인 네트워크를 식별하는 기본 정보
컨트롤러 연결 자원 구성을 확인하는 기준 정보
네트워크 대역 현재 네트워크의 주소 범위를 확인하는 항목

▶처리 결과 / 유의사항:

네트워크 소유자는 네트워크 수정, 삭제, 장치 생성, 수정, 삭제, 사용자 초대가 가능하다.

장치 사용자는 본인 장치 별칭 수정이 가능하다.

일반 참여자는 조회 위주로 사용한다.

검색 기준으로 네트워크 이름, 컨트롤러, 네트워크 대역을 사용할 수 있다.

상세 화면에서는 장치 상태, 사용자 할당, 단말 접속 허용, 외부 트래픽 허용 여부를 함께 확인한다.

새 네트워크 생성 전에는 함께 사용할 멤버를 먼저 정리하면 작업이 빠르다.

네트워크 작업 후에는 목록 반영 여부와 상세 화면의 장치 상태 변화를 함께 확인해야 정상이다.

▶스크린샷:

화면명 이미지
네트워크 화면

▶추가 화면:

화면명 이미지
새 네트워크 생성 팝업
네트워크 상세 화면

5.3 멤버

▶메뉴 위치: 상단 메뉴 > 멤버

▶기능 설명:

조직 트리 기준으로 멤버를 조회한다.

이름, 아이디, 연락처, 이메일 기준 검색이 가능하다.

▶사용 절차:

1. 멤버 메뉴를 선택한다.

2. 좌측 조직 트리에서 부서를 선택한다.

3. 검색 조건을 입력한다.

4. 결과 목록에서 멤버 정보를 확인한다.

▶입력 항목:

항목 설명 입력 가이드
검색 조건 조회 기준 아이디 또는 이름 등
검색어 멤버 조회 키워드 부분 검색 가능

▶화면 내 주요 버튼:

버튼명 설명
검색 검색 조건에 따라 멤버를 조회한다.

▶처리 결과 / 유의사항:

선택한 부서의 하위 부서 구성원까지 함께 조회된다.

멤버 초대 대상을 찾을 때는 부서 트리와 검색 조건을 함께 사용하는 것이 효율적이다.

네트워크에 초대할 대상을 찾을 때 가장 먼저 확인하면 좋다.

▶스크린샷:

화면명 이미지
멤버 화면

5.4 요청

▶메뉴 위치: 상단 메뉴 > 요청

▶기능 설명:

받은 요청과 보낸 요청을 분리해 조회한다.

상태별로 전체, 대기, 승인, 반려 요청을 확인한다.

받은 요청 중 승인 권한이 있는 경우 처리 기능이 제공될 수 있다.

같은 요청을 다시 보내기 전에 현재 상태를 먼저 확인하는 화면이다.

▶사용 절차:

1. 요청 메뉴를 선택한다.

2. `받은 요청` 또는 `보낸 요청` 탭을 선택한다.

3. 상태 필터를 선택한다.

4. 요청 카드를 확인한다.

5. 권한이 있는 경우 요청을 승인 또는 반려한다.

▶처리 결과 / 유의사항:

승인/반려 기능은 결재 권한이 있는 사용자에게만 적용될 수 있다.

실시간 이벤트 수신 시 목록이 갱신된다.

`받은 요청` 탭에서는 대기 요청을 처리하고 `보낸 요청` 탭에서는 현재 승인 상태를 확인한다.

반려된 요청은 반려 사유를 확인한 뒤 내용을 수정해 다시 요청해야 한다.

`검토 중` 상태에서 다시 같은 요청을 보내면 중복 작업이 되므로 먼저 기존 요청의 진행 상태를 확인한다.

상태 배지와 하단 안내 문구를 함께 읽으면 승인 완료, 검토 중, 반려 사유를 빠르게 구분할 수 있다.

▶스크린샷:

화면명 이미지
요청 화면

▶추가 화면:

화면명 이미지
보낸 요청 화면

5.5 설정

▶메뉴 위치: 상단 메뉴 > 설정

▶기능 설명:

프로필 수정, 비밀번호 변경, 계정 삭제를 수행한다.

개인 정보와 접속 보안을 정리하는 마지막 점검 화면으로 이해하면 된다.

▶사용 절차:

1. 설정 메뉴를 선택한다.

2. 프로필 탭에서 이름, 연락처, 이메일을 수정한다.

3. 보안 탭에서 현재 비밀번호와 새 비밀번호를 입력한다.

4. 계정 삭제 탭에서 삭제 여부를 확인한다.

▶입력 항목:

항목 설명 입력 가이드
이름 사용자 이름 2자 이상
휴대폰 번호 연락처 `010-0000-0000` 형식
이메일 이메일 주소 형식 검증 적용
현재 비밀번호 기존 비밀번호 8~16자 정책
새 비밀번호 변경 비밀번호 8~16자 정책
새 비밀번호 확인 변경 비밀번호 확인 동일 값 입력

▶화면 내 주요 버튼:

버튼명 설명
변경사항 저장 프로필 정보를 저장한다.
비밀번호 변경 비밀번호를 변경한다.
계정 삭제 계정을 삭제한다.

▶처리 결과 / 유의사항:

비밀번호는 영문, 숫자, 특수문자 포함 8~16자 검증이 적용된다.

계정 삭제는 되돌릴 수 없는 작업으로 확인된다.

설정 화면은 프로필, 보안, 계정 삭제 탭으로 구성된다.

현재 비밀번호가 맞지 않거나 형식 기준을 만족하지 못하면 비밀번호가 저장되지 않는다.

프로필 변경 후에는 저장 완료 메시지와 최신 연락처 반영 여부를 함께 확인한다.

6. 일일 사용 체크리스트

6.1 매일 확인

홈 알림과 최근 요청이 있는지 본다.

참여 중인 네트워크 수와 장치 온라인/오프라인 상태를 확인한다.

요청 화면에서 대기 또는 검토 중 상태가 남아 있는지 확인한다.

6.2 작업 전 확인

초대할 사용자와 작업할 네트워크가 맞는지 먼저 확인한다.

같은 요청을 이미 보낸 적이 없는지 `보낸 요청` 탭을 먼저 확인한다.

연락처와 이메일 등 개인 정보가 최신 상태인지 확인한다.

6.3 작업 후 확인

네트워크 목록 또는 상세 화면에 변경 내용이 반영됐는지 확인한다.

장치 상태가 기대한 값으로 보이는지 확인한다.

요청 상태와 설정 저장 완료 메시지를 다시 확인한다.

7. 자주 하는 실수와 대응

7.1 요청이 처리되지 않았는데 다시 같은 요청을 보내는 경우

먼저 `보낸 요청` 탭에서 현재 상태가 `검토 중`인지, 이미 `반려`됐는지 확인한다.

7.2 장치가 오프라인인데 네트워크 생성 문제로 오해하는 경우

홈과 네트워크 상세 화면에서 같은 장치 상태를 다시 확인하고, 실제 단말 연결 상태도 함께 점검한다.

7.3 비밀번호가 저장되지 않는 경우

현재 비밀번호 입력 여부, 새 비밀번호 형식, 새 비밀번호 확인 값 일치를 순서대로 확인한다.

7.4 반려 사유를 읽지 않고 다시 요청하는 경우

반려 사유를 먼저 읽고 내용을 수정한 뒤 다시 요청해야 같은 반려를 반복하지 않는다.

## 1. 문서 안내

이 문서는 IDN 2.0 최고 관리자가 실제 운영 화면에서 업무를 수행할 수 있도록 만든 사용자 매뉴얼이다.  
메뉴 설명만 나열하지 않고, 다음 순서로 바로 따라 할 수 있게 구성했다.

1. 어디로 들어가는지
2. 화면에서 무엇을 할 수 있는지
3. 버튼을 어떤 순서로 눌러야 하는지
4. 각 입력 항목이 무엇을 뜻하는지
5. 작업 전후로 무엇을 확인해야 하는지

이 문서는 다음 사용자를 대상으로 한다.

- 최고 관리자 업무를 처음 맡은 운영자
- 기존 운영자를 인수인계받는 관리자
- 실제 화면 기준 운영 절차서를 준비해야 하는 조직

---

## 2. 최고 관리자 역할

최고 관리자는 시스템 전체 범위의 운영 권한을 가진다.

- 전체 대시보드 확인
- 컨트롤러, 중계서버, VNET, 엔드포인트 관리
- 사용자, 조직, 부서, 직위 관리
- 보안 정책 설정
- 시험 파일 업로드와 시스템 품질 시험 실행
- 관리자 정보 및 접속 장치 관리
- 감사 로그 조회와 백업 다운로드

실무에서는 다음 순서로 운영하는 것이 가장 안전하다.

1. 대시보드에서 현황과 이상 징후를 먼저 본다.
2. 조직, 부서, 직위, 사용자를 정비한다.
3. 컨트롤러, 중계서버, VNET, 엔드포인트를 관리한다.
4. 보안 정책과 관리자 접근 정책을 점검한다.
5. 시스템 품질 시험으로 변경 결과를 검증한다.
6. 감사 로그에서 작업 이력을 확인한다.

## 3. 운용 전 확인 사항

로그인 전에 다음 항목을 먼저 확인한다.

- 접속 URL이 정확한지 확인한다.
- 관리자 계정이 잠금 상태가 아닌지 확인한다.
- 최고 관리자 접속 장치 IP가 허용 대상인지 확인한다.
- 브라우저 팝업 차단이 과도하지 않은지 확인한다.
- 라이선스 파일, 시험 파일처럼 업로드가 필요한 자료는 미리 준비한다.

권장 환경은 다음과 같다.

- 최신 Chromium 계열 브라우저 사용
- 화면 배율 100% 권장
- 장시간 작업 시 새로고침보다 화면 내 검색과 필터를 우선 사용

## 4. 로그인

### 4.1 로그인 방법

1. 브라우저에서 `https://61.109.220.75:15881/`에 접속한다.
2. 아이디와 비밀번호를 입력한다.
3. 로봇 확인 항목을 완료한다.
4. `로그인` 버튼을 클릭한다.

![로그인](manual-screenshots/login.png)

### 4.2 로그인 화면 항목

- 아이디: 관리자 계정 ID
- 비밀번호: 관리자 계정 비밀번호
- 로봇 확인: 자동 로그인 차단을 위한 검증
- 로그인 버튼: 인증 후 대시보드로 이동

### 4.3 로그인 시 알아둘 점

- 최고 관리자 로그인 직후 내부 동기화로 첫 화면 전환이 늦을 수 있다.
- 비밀번호 정책에 따라 로그인 후 비밀번호 변경이 요구될 수 있다.
- 장시간 무작동 시 자동 로그아웃될 수 있다.

### 4.4 로그인 실패 시 점검 항목

- 아이디 오입력 여부
- 비밀번호 오입력 여부
- 접근 허용 장치 IP 등록 여부
- 로그인 실패 누적에 따른 계정 잠금 여부
- 비밀번호 만료 여부

## 5. 공통 화면 사용법

### 5.1 상단 헤더

상단 헤더에서는 다음 정보를 확인하거나 사용할 수 있다.

- 비밀번호 만료 예정일
- 알림 아이콘
- 도움말 또는 설정 아이콘
- 프로필 메뉴

### 5.2 좌측 메뉴

최고 관리자 기준 주요 메뉴는 다음과 같다.

- 대시보드
- 관리
- 보안 및 정책
- 시스템품질
- 관리자 설정
- 감사

`관리` 하위 메뉴는 다음으로 구성된다.

- 컨트롤러 관리
- 중계서버 관리
- 가상보안네트워크 관리
- 사용자 관리
- 조직 관리
- 부서 관리
- 알람 관리
- 이상감지 관리
- 라이선스 관리
- 요청 관리

### 5.3 자주 사용하는 화면 조작

- 검색창: 원하는 대상을 빠르게 찾을 때 사용한다.
- 필터: 상태, 유형, 결과처럼 범위를 좁힐 때 사용한다.
- 테이블 행 액션: 수정, 삭제, 승인, 반려 같은 개별 작업에 사용한다.
- 모달 팝업: 생성, 수정, 업로드, 확인 작업에 사용한다.
- 사이드 패널: 상세 정보나 부가 정보를 우측에서 확인할 때 사용한다.

### 5.4 최고 관리자에게 중요한 공통 규칙

- `전체` 범위에서는 생성이 막히는 메뉴가 있다.
- 컨트롤러, 중계서버, VNET, 엔드포인트, 사용자 생성 전에는 상단 그룹 선택에서 특정 그룹을 먼저 선택해야 한다.
- 삭제 작업 전에는 영향 범위를 먼저 확인한다.
- 중요한 변경 후에는 시스템 품질과 감사 로그를 함께 확인한다.

## 6. 대시보드

### 6.1 진입 경로

- 좌측 메뉴 `대시보드`
- 직접 URL: `/system/dashboard`

### 6.2 대시보드에서 확인하는 정보

- 알림 현황
- 총 사용자 수
- 접속 사용자 수
- 차단 사용자 수
- 등록 사용자 수
- VNET별 최근 접속 사용자 수
- 리소스 현황
- 최근 감사 이력

### 6.3 대시보드 활용 방법

- 접속 사용자 수가 급감하면 사용자, VNET, 엔드포인트, 컨트롤러 상태를 함께 본다.
- 차단 사용자 수가 급증하면 보안 정책과 IP 차단 정책을 확인한다.
- 최근 감사 이력에서 누가 어떤 작업을 했는지 먼저 확인한다.

### 6.4 운영 팁

- 대시보드는 설정 화면이 아니라 점검 시작 화면으로 사용한다.
- 이상 징후를 발견하면 세부 조치는 하위 관리 메뉴에서 수행한다.

![대시보드](manual-screenshots/dashboard.png)

## 7. 관리 메뉴

## 7.1 컨트롤러 관리

### 7.1.1 진입 경로

- 좌측 메뉴 `관리 > 컨트롤러 관리`
- 직접 URL: `/system/management/controller`

### 7.1.2 이 화면에서 하는 일

- 컨트롤러 목록 조회
- 검색
- 등록
- 수정
- 삭제
- 연결 상태와 등록 상태 확인

### 7.1.3 먼저 봐야 하는 컬럼

- 컨트롤러 이름
- 외부 IP / 외부 Port
- 내부 IP / 내부 Port
- 상태
- 등록 상태
- 생성일
- 설명

### 7.1.4 검색 방법

1. 검색창에서 검색 기준을 선택한다.
2. 컨트롤러 이름 또는 ID 성격의 값을 입력한다.
3. Enter로 조회한다.

### 7.1.5 컨트롤러 등록 방법

1. 상단 그룹 선택에서 특정 그룹을 선택한다.
2. `컨트롤러 등록` 버튼을 클릭한다.
3. `컨트롤러 이름`, `설명`을 입력한다.
4. `등록하기`를 클릭한다.
5. 목록에서 생성 결과를 확인한다.

등록 팝업 입력 항목

- 컨트롤러 이름: 컨트롤러 식별자 역할의 필수값
- 설명: 운영 메모, 설치 위치, 용도를 적는 선택값

사용자 안내

- 그룹을 선택하지 않으면 `"컨트롤러 그룹 미선택"` 안내가 표시된다.
- 실제 팝업 버튼은 `닫기`, `등록하기`다.

![컨트롤러 등록 팝업](manual-modal-screenshots/controller-create.png)

### 7.1.6 컨트롤러 수정 방법

1. 대상 행의 `수정` 액션을 클릭한다.
2. 기본 정보 또는 `고급 설정`의 네트워크 정보를 수정한다.
3. 저장 후 상태 변화를 확인한다.

수정 시 확인 항목

- 외부 IP
- 외부 Port
- 내부 IP
- 내부 Port
- 네트워크 인터페이스
- UUID
- MAC Address

### 7.1.7 주의사항

- 잘못된 IP, 포트 변경은 VNET과 엔드포인트 운영에 직접 영향을 준다.
- 변경 후에는 연결 상태와 관련 감사 로그를 함께 확인한다.

![컨트롤러 관리](manual-screenshots/controller.png)

## 7.3 가상보안네트워크 관리

가상보안네트워크 관리는 `VNET`, `엔드포인트`, `토폴로지` 3개 화면으로 구성된다.

## 7.3.1 VNET 관리

### 7.3.1.1 진입 경로

- 좌측 메뉴 `관리 > 가상보안네트워크 관리`
- 직접 URL: `/system/management/network/vnet`

### 7.3.1.2 이 화면에서 하는 일

- VNET 목록 조회
- 검색
- 생성
- 수정
- 삭제
- 접속 정책과 외부 트래픽 정책 확인

### 7.3.1.3 먼저 이해해야 하는 항목

- VNET ID: 시스템에서 VNET을 식별하는 값
- 별칭: 운영자가 읽는 표시명
- 컨트롤러: 해당 VNET을 제어하는 장비
- 중계서버: 필요한 경우 연결되는 중계 장비
- CIDR: 네트워크 주소 범위
- 단말 접속 허용: 엔드포인트 접속 가능 여부
- 외부 트래픽 허용: VNET 외부 통신 허용 여부
- 기기 인증: 보안 장치 사용 여부
- 삭제예정일: 유예 삭제 기준 날짜

### 7.3.1.4 VNET 생성 방법

1. 상단 그룹 선택에서 특정 그룹을 선택한다.
2. `VNET 생성` 버튼을 클릭한다.
3. VNET ID와 별칭을 입력한다.
4. 컨트롤러를 선택한다.
5. 필요 시 중계서버를 선택한다.
6. 암호화 방식과 주소 대역을 설정한다.
7. 접속 정책과 삭제예정일을 입력한다.
8. 저장 후 목록에 반영되었는지 확인한다.

### 7.3.1.5 입력 가이드

- VNET ID: 필수, 운영 식별자로 계속 사용되므로 임시 이름 사용 금지
- 별칭: 운영자 표시명
- 컨트롤러: 필수, 잘못 선택하면 엔드포인트 운영이 꼬일 수 있음
- 중계서버: 필요한 경우만 선택
- 암호화 알고리즘: `ARIA_256`, `AES_256`, `LEA_256`
- 암호화 모드: `CBC`, `GCM`
- CIDR 길이: `/8`, `/16`, `/24`
- 단말 접속 허용: 엔드포인트 접속 허용 여부
- 외부 트래픽 허용: 외부망 통신 허용 여부
- 기기 인증 여부: 추가 장치 인증 사용 여부
- 삭제예정일: 필수

사용자 안내

- 그룹을 선택하지 않으면 `"VNET 그룹 미선택"` 안내가 표시된다.

## 7.3.2 엔드포인트 관리

### 7.3.2.1 진입 경로

- 네트워크 관리 상단 탭 `엔드포인트`
- 직접 URL: `/system/management/network/endpoint`

### 7.3.2.2 이 화면에서 하는 일

- 엔드포인트 목록 조회
- 생성
- 수정
- 삭제
- 단말 접속 허용/차단
- 외부 트래픽 허용/차단
- 사용자 할당/해제
- 메모 확인

### 7.3.2.3 먼저 이해해야 하는 항목

- 엔드포인트 ID
- 별칭
- 역할
- TNI Address
- MAC Address
- 단말 접속
- 외부 트래픽
- 상태
- 사용자할당

### 7.3.2.4 엔드포인트 생성 방법

1. 상단 그룹 선택에서 특정 그룹을 선택한다.
2. `엔드포인트 생성` 버튼을 클릭한다.
3. 소속 VNET을 선택한다.
4. 엔드포인트 ID와 별칭을 입력한다.
5. 역할을 선택한다.
6. TNI 주소와 정책을 설정한다.
7. 필요 시 리다이렉트 정보를 추가한다.
8. 저장 후 목록에 반영되었는지 확인한다.

### 7.3.2.5 입력 가이드

- VNET: 소속 네트워크
- 엔드포인트 ID: 필수 식별값
- 엔드포인트 별칭: 운영자 표시명
- 역할: `HOST`, `GATEWAY`
- TNI Address: 가상 네트워크 주소
- CIDR 길이: `/8`, `/16`, `32`
- 리다이렉트 IP 추가: 게이트웨이 운영 시 추가 주소 입력
- 단말 접속 허용: 접속 가능 여부
- 외부 트래픽 허용: 외부망 통신 가능 여부
- 네트워크 인터페이스 / MAC 주소 / UUID: 역할과 자동 할당 상태에 따라 활성화

사용자 안내

- 그룹을 선택하지 않으면 `"엔드포인트 그룹 미선택"` 안내가 표시된다.

![엔드포인트 생성 팝업](manual-modal-screenshots/endpoint-create.png)
![엔드포인트 관리](manual-screenshots/endpoint.png)

### 7.3.2.6 사용자 할당 방법

1. 대상 엔드포인트의 사용자 할당 기능을 연다.
2. ID, 이름, 휴대폰번호, 이메일 기준으로 사용자를 검색한다.
3. 사용자를 선택한다.
4. 저장 후 `사용자할당` 컬럼을 확인한다.

### 7.3.2.7 운영 팁

- HOST 역할은 실제 사용자 할당 상태를 함께 관리해야 한다.
- GATEWAY 역할은 리다이렉트 정보가 정확한지 반드시 확인한다.

### 7.5.1 진입 경로

### 7.5.2 이 화면에서 하는 일

- 즉시 삭제

### 7.5.3 조직 카드에서 확인하는 정보

- 조직명

- 현재 VNET 수 / 최대 VNET 수
- 현재 엔드포인트 수 / 최대 엔드포인트 수
- 삭제 예정일
- 생성 상태

### 7.5.4 조직 생성 흐름

조직 생성은 4단계 마법사 구조다.

1. 기본 정보
2. 리소스 설정
3. 관리자 계정
4. 사용자 선택

### 7.5.5 단계별 입력 의미

1단계 기본 정보

- 회사 선택: 조직명과 그룹 기본값의 기준
- 설명: 조직 운영 목적, 고객사, 환경 설명

2단계 리소스 설정

- 최대 유저 수
- 최대 VNET 수
- 최대 엔드포인트 수

3단계 관리자 계정

- 관리자 아이디
- 관리자 비밀번호
- 비밀번호 확인

4단계 사용자 선택

- 실제 그룹 관리자 계정을 운영할 사용자 지정

### 7.5.6 사용자 안내

- 현재 운영 서버에서는 조직 관리 페이지 렌더링이 일시적으로 불안정할 수 있었다.
- 다만 소스코드 기준으로는 위 4단계를 모두 완료해야 최종 생성이 가능하다.

### 7.5.7 삭제 방식 이해

- 유예 삭제: 일정 기간 후 자동 삭제, 복구 가능
- 즉시 삭제: 즉시 영구 삭제, 복구 불가
- 유예 삭제 취소: 삭제 예정 상태 해제

### 7.5.8 운영 팁

- 운영 중 조직은 즉시 삭제보다 유예 삭제를 우선 사용한다.
- 삭제 전에는 하위 사용자와 네트워크 자원 영향을 먼저 점검한다.

### 8.4.3 시험 파일 업로드 방법

1. 상단 탭에서 대상 유형을 선택한다.
2. `시험 파일 업로드` 버튼을 클릭한다.
3. 파일을 드래그하거나 선택한다.
4. 업로드 목록을 확인한다.
5. `업로드하기`를 클릭한다.

입력 의미

- 대상 유형: 현재 탭 기준으로 컨트롤러 또는 클라이언트에 연결
- 파일 목록: 업로드 예정 파일
- 중복 표시: 동일 파일명 또는 중복 파일 확인용

사용자 안내

- 팝업 제목은 `시험 파일 업로드`다.
- 업로드 영역에는 파일 드래그 안내가 표시된다.
- 실제 버튼은 `취소`, `업로드하기`다.

![시험 파일 업로드 팝업](manual-modal-screenshots/test-file-upload.png)
![시험 파일 관리](manual-screenshots/test-file.png)

### 8.4.4 운영 팁

- 시스템 품질 시험 전에 필요한 시험 파일이 미리 등록되어 있는지 확인한다.
- 해시값은 파일 동일성 검증에 사용한다.

## 9. 시스템 품질

### 9.1 진입 경로

- 좌측 메뉴 `시스템품질`
- 직접 URL: `/system/quality`

### 9.2 이 화면에서 하는 일

- 시험 이력 조회
- 기간과 필터 검색
- 수동 시험 시작
- 정기 시험 예약

### 9.3 시험 시작 방법

1. `시험 시작` 버튼을 클릭한다.
2. 시험 대상을 선택한다.
3. 필요한 추가 선택을 진행한다.
4. `시험 시작`을 클릭한다.

시험 대상

- 오케스트레이터
- DB
- 컨트롤러
- 클라이언트

사용자 안내

- 팝업 문구는 `시험을 실시할 대상을 선택하십시오.`다.
- 실제 버튼은 `취소`, `시험 시작`이다.

![시험 시작 팝업](manual-modal-screenshots/quality-start-test.png)

### 9.4 예약 설정 방법

1. `예약 설정` 버튼을 클릭한다.
2. 반복 주기를 선택한다.
3. 요일과 시간을 설정한다.
4. `저장`을 클릭한다.

입력 가이드

- 반복: `매일`, `매주`, `매월`
- 요일: `월`, `화`, `수`, `목`, `금`, `토`, `일`
- 시간: `AM/PM`, 시 `01~12`, 분 `00/30`

![예약 설정 팝업](manual-modal-screenshots/quality-schedule.png)
![시스템 품질](manual-screenshots/quality.png)

### 9.5 운영 팁

- 변경 작업 직후에는 수동 시험으로 먼저 확인한다.
- 정기 시험은 저부하 시간대로 예약한다.
- 실패 시 알람 관리와 감사 로그를 함께 본다.

## 10. 관리자 설정

### 10.1 진입 경로

- 좌측 메뉴 `관리자 설정`
- 직접 URL: `/system/account`

### 10.2 이 화면에서 하는 일

- 관리자 기본 정보 수정
- 비밀번호 변경
- 전체 동기화
- 관리자 접속 장치 등록/수정/삭제

### 10.3 비밀번호 변경 방법

1. `비밀번호 변경` 버튼을 클릭한다.
2. 현재 비밀번호, 새 비밀번호, 새 비밀번호 확인을 입력한다.
3. `변경`을 클릭한다.
4. 로그아웃 후 다시 로그인한다.

입력 기준

- 영문, 숫자, 특수문자를 포함한 8~16자
- 새 비밀번호와 확인 비밀번호는 같아야 한다.

사용자 안내

- 비밀번호 변경 시 보안을 위해 로그아웃된다.
- 실제 버튼은 `취소`, `변경`이다.

![비밀번호 변경 팝업](manual-modal-screenshots/account-change-password.png)

### 10.4 관리자 장치 등록 방법

1. `장치 등록` 버튼을 클릭한다.
2. 장치 이름, 장치 IP, 장치 설명을 입력한다.
3. `등록하기`를 클릭한다.

입력 가이드

- 장치 이름: 접속 장치를 식별할 이름
- 장치 IP: 허용할 관리자 장치 IP
- 장치 설명: 위치, 담당자, 사용 목적

사용자 안내

- 실제 팝업 제목은 `관리자 장치 등록`이다.
- 실제 버튼은 `취소`, `등록하기`다.

![관리자 장치 등록 팝업](manual-modal-screenshots/account-terminal-create.png)
![관리자 설정](manual-screenshots/account.png)

### 10.5 운영 팁

- 등록된 관리자 장치만 접근 허용 정책에 반영될 수 있으므로 실제 사용하는 단말 위주로 등록한다.
- 대량 변경 후 상태가 맞지 않으면 `전체 동기화`를 검토한다.

## 12. 최고 관리자 일일 운영 체크리스트

### 12.1 시작 점검

1. 대시보드에서 사용자 수, 알람, 최근 감사 이력을 확인한다.
2. 알람 관리에서 미해결 항목이 있는지 확인한다.
3. 시스템 품질 최근 시험 결과를 확인한다.

### 12.2 변경 작업 전 점검

1. 라이선스 여유를 확인한다.
2. 최근 감사 로그에서 선행 변경 이력을 확인한다.
3. 영향 대상 조직, 부서, VNET, 엔드포인트를 정리한다.

### 12.3 변경 작업 후 점검

1. 해당 메뉴 목록에서 저장 결과를 확인한다.
2. 대시보드 또는 상세 화면에서 상태 변화를 확인한다.
3. 감사 로그에 기록이 남았는지 확인한다.
4. 필요 시 시스템 품질 시험을 수동 실행한다.

## 13. 자주 하는 실수와 대응

### 13.1 자주 하는 실수

- 사용자 생성 전에 부서와 직위를 만들지 않는 경우
- VNET 생성 시 잘못된 컨트롤러를 선택하는 경우
- 시험 파일을 먼저 등록하지 않고 품질 시험을 시작하는 경우
- 요청 반려 사유를 모호하게 쓰는 경우
- 관리자 장치 IP를 등록하지 않아 로그인에 문제가 생기는 경우
- 조직을 즉시 삭제해 복구가 불가능해지는 경우

### 13.2 대응 원칙

- 구조 정보는 `조직 > 부서 > 직위 > 사용자` 순으로 먼저 만든다.
- 네트워크 자원은 `컨트롤러/중계서버 > VNET > 엔드포인트` 순으로 만든다.
- 삭제는 즉시 삭제보다 유예 삭제를 우선 검토한다.
- 정책 변경 후에는 알람, 시스템 품질, 감사 로그를 함께 확인한다.

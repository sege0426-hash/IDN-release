## 1. 문서 안내

이 문서는 IDN 2.0 최고 관리자가 실제 운영 화면에서 서비스 전체를 운용할 수 있도록 만든 사용자 가이드이다.

최고 관리자는 시스템 전 범위를 다루기 때문에 단순 메뉴 소개만으로는 업무 수행이 어렵다.  
따라서 이 문서는 실제 운영자가 화면을 보면서 다음 질문에 바로 답할 수 있게 구성했다.

1. 이 메뉴는 언제 들어가는가
2. 화면에서 가장 먼저 무엇을 보는가
3. 어떤 버튼으로 작업을 시작하는가
4. 각 필드는 무엇을 의미하는가
5. 저장 후 무엇을 확인해야 하는가

추천 읽기 순서

1. 로그인과 화면 구조
2. 운용 시작 절차
3. 조직과 사용자 구조 정비
4. 네트워크와 장치 운영
5. 보안 정책과 품질 점검
6. 감사 확인

---

## 2. 최고 관리자 역할과 운영 원칙

최고 관리자는 시스템 전체 범위의 운영 권한을 가진다.

주요 업무

- 전체 대시보드 확인
- 컨트롤러, 중계서버, VNET, 엔드포인트 관리
- 사용자, 조직, 부서, 직위 관리
- 보안 및 정책 관리
- 라이선스 업데이트
- 시스템 품질 시험 실행과 예약
- 관리자 정보 및 접속 장치 관리
- 감사 로그 조회와 백업 다운로드

실무 운영 원칙

1. 먼저 현황을 보고, 그 다음에 수정한다.
2. 조직과 사용자 구조를 먼저 맞추고 자원을 만든다.
3. 변경 작업 전에는 영향을 받는 그룹과 네트워크를 확인한다.
4. 저장 후에는 목록 반영, 상태 변화, 감사 이력을 반드시 확인한다.
5. 보안 정책과 품질 시험은 운영 변경 후 함께 점검한다.

## 3. 로그인과 화면 구조

### 3.1 로그인 방법

1. 브라우저에서 배포된 `IDN 운영 포털 주소`에 접속한다.
2. 아이디와 비밀번호를 입력한다.
3. 로봇 확인 항목을 완료한다.
4. `로그인` 버튼을 클릭한다.

![로그인](manual-screenshots/login.png)

### 3.2 로그인 후 보이는 전체 메뉴

최고 관리자 화면은 좌측 사이드바 중심의 관리자 콘솔로 구성된다.

상위 메뉴

- 대시보드
- 관리
- 보안 및 정책
- 시스템품질
- 관리자 설정
- 감사

`관리` 하위 메뉴

- 컨트롤러 관리
- 중계서버 관리
- 가상보안네트워크 관리
- 사용자 관리
- 조직 관리
- 부서 관리
- 알람 관리
- 이상감지 관리
- 라이선스 관리
- 요청 관리

### 3.3 최고 관리자 화면에서 꼭 이해해야 할 개념

최고 관리자는 `전체` 범위와 `특정 그룹` 범위를 구분해서 작업한다.

- 전체 범위: 전체 현황 조회와 전사 정책 관리에 사용
- 특정 그룹 범위: 컨트롤러, 중계서버, VNET, 엔드포인트, 사용자 같은 생성 작업에 사용

실제 운영에서는 다음 규칙을 기억하면 된다.

- 조회는 `전체`에서 시작해도 된다.
- 생성과 수정은 대상 그룹을 먼저 정한 뒤 진행하는 것이 안전하다.
- 그룹 선택이 되지 않은 상태에서는 일부 등록 버튼이 동작하지 않거나 안내 문구가 먼저 표시된다.

## 4. 최고 관리자 운용 시작 절차

하루 업무를 시작할 때는 다음 순서로 점검한다.

1. `대시보드`에서 전체 상태와 이상 징후를 확인한다.
2. `요청 관리`에서 대기 요청을 먼저 확인한다.
3. `조직 관리`, `부서 관리`, `사용자 관리`에서 구조 변경 필요 여부를 확인한다.
4. 대상 그룹을 선택한 뒤 `컨트롤러`, `중계서버`, `VNET`, `엔드포인트`를 점검한다.
5. `보안 및 정책`에서 차단 정책과 시험 파일 상태를 확인한다.
6. 변경이 있었다면 `시스템품질`에서 시험 또는 예약을 확인한다.
7. 마지막으로 `감사`에서 주요 변경 이력을 확인한다.

## 6.3 가상보안네트워크 관리

진입 경로

- `관리 > 가상보안네트워크 관리`

할 수 있는 작업

- VNET 조회
- VNET 생성, 수정, 삭제
- 엔드포인트 생성과 수정
- 토폴로지 확인
- 네트워크 접근 상태 확인

![VNET 관리](manual-screenshots/vnet.png)

### VNET 생성 절차

1. 대상 그룹을 선택한다.
2. `생성` 버튼을 클릭한다.
3. ID와 별칭을 입력한다.
4. 컨트롤러와 필요 시 중계서버를 선택한다.
5. 암호화 알고리즘과 모드를 선택한다.
6. IP 대역과 길이를 입력한다.
7. 삭제 날짜와 보안 옵션을 설정한다.
8. `생성하기`를 클릭한다.

입력 항목 의미

- ID: VNET 고유 식별값
- 별칭: 화면 표시용 이름
- 컨트롤러: 연결을 제어할 장비
- 중계서버: 필요한 경우 경유 장비
- 알고리즘 / 모드: 암호화 방식
- IP 대역 / 길이: 네트워크 주소 범위
- 삭제 날짜: 자동 정리 기준일
- 단말 접속 허용: 엔드포인트 접속 허용 여부
- 외부 트래픽 허용: 외부망 통신 허용 여부
- 기기 인증 여부: 인증 사용 여부

운영 팁

- 이미 존재하는 대역과 겹치면 생성되지 않는다.
- 운영자가 읽기 쉬운 별칭을 사용하면 요청과 감사에서 확인이 편하다.

![VNET 생성](manual-modal-screenshots/vnet-create.png)

### 엔드포인트 생성 절차

1. 엔드포인트 화면 또는 토폴로지에서 `생성`을 클릭한다.
2. VNET을 선택한다.
3. 엔드포인트 ID와 별칭을 입력한다.
4. 역할과 네트워크 주소를 설정한다.
5. 필요 시 인터페이스, MAC 주소, UUID를 입력한다.
6. 접속 허용과 트래픽 허용 여부를 설정한다.
7. 저장한다.

입력 항목 의미

- VNET: 소속 네트워크
- 엔드포인트 ID: 장치 식별값
- 엔드포인트 별칭: 화면 표시명
- 역할: 장치 역할
- 네트워크 주소: 할당 주소
- 네트워크 인터페이스: 실제 장치 인터페이스명
- MAC 주소: 장치 식별용 값
- UUID: 장치 고유 식별값

1. 대상 그룹을 선택한다.
2. `등록` 버튼을 클릭한다.
3. 아이디, 이름, 이메일, 휴대폰번호를 입력한다.

## 12. 자주 발생하는 상황과 대응

### 12.1 등록 버튼을 눌렀는데 안내만 보이는 경우

- 현재 `전체` 범위일 수 있다.
- 컨트롤러, 중계서버, VNET, 엔드포인트, 사용자 생성은 대상 그룹을 먼저 정해야 한다.

### 12.2 저장은 됐는데 목록에서 바로 보이지 않는 경우

- 검색 조건이나 필터가 남아 있는지 확인한다.
- 동기화가 필요한 작업인지 확인한다.
- 같은 이름의 다른 항목과 혼동하지 않았는지 확인한다.

### 12.3 VNET 생성이 실패하는 경우

- IP 대역이 기존 네트워크와 겹치지 않는지 확인한다.
- 컨트롤러와 중계서버가 올바른 그룹 범위에 있는지 확인한다.

### 12.4 요청 처리 후 사용자 문의가 계속 들어오는 경우

- 승인만 됐고 상위 결재가 남아 있는지 확인한다.
- 최종 반영 후 네트워크, 장치, 사용자 상태를 다시 확인한다.

## 1. 문서 안내

이 문서는 IDN 2.0 그룹 관리자가 자기 조직 범위 안에서 필요한 업무를 수행할 수 있도록 만든 사용자 매뉴얼이다.

그룹 관리자는 최고 관리자와 달리 시스템 전체를 관리하지 않는다.  
따라서 이 문서는 다음 두 가지를 분명하게 구분해서 설명한다.

1. 그룹 관리자가 할 수 있는 업무
2. 그룹 관리자가 할 수 없는 업무

문서는 실제 사용자 관점에서 다음 순서로 구성했다.

1. 어디로 들어가는지
2. 화면에서 무엇을 할 수 있는지
3. 버튼을 어떤 순서로 눌러야 하는지
4. 각 항목이 무엇을 의미하는지
5. 작업 후 무엇을 확인해야 하는지

---

## 2. 그룹 관리자 역할

그룹 관리자는 자기 그룹 범위의 운영 권한을 가진다.

- 그룹 범위 대시보드 확인
- 컨트롤러, 중계서버, VNET, 엔드포인트 관리
- 그룹 소속 사용자 관리
- 그룹 내 부서와 직위 관리
- 알람과 이상감지 기준 확인
- 그룹 범위 요청 승인과 반려
- 관리자 정보 수정
- 그룹 데이터 동기화
- 그룹 범위 감사 로그 조회

그룹 관리자에게 없는 권한은 다음과 같다.

- 전체 조직 생성, 수정, 삭제
- 전체 라이선스 관리
- 보안 및 정책 메뉴 관리
- 시스템 품질 메뉴 관리
- 전체 시스템 감사 범위 관리
- 최고 관리자 접속 장치 관리

실무에서는 다음 순서로 운영하는 것이 좋다.

1. 대시보드에서 현재 그룹 상태를 확인한다.
2. 부서, 직위, 사용자 구조를 먼저 정리한다.
3. 컨트롤러, 중계서버, VNET, 엔드포인트를 관리한다.
4. 요청 처리와 알람을 점검한다.
5. 필요 시 그룹 데이터 동기화를 수행한다.
6. 감사 로그에서 작업 이력을 확인한다.

## 3. 운용 전 확인 사항

로그인 전에 다음 항목을 확인한다.

- 접속 URL이 정확한지 확인한다.
- 그룹 관리자 계정이 잠금 상태가 아닌지 확인한다.
- 비밀번호 만료 여부를 확인한다.
- 브라우저 팝업 차단이 과도하지 않은지 확인한다.
- 운영에 필요한 업로드 자료나 변경 대상 목록을 미리 준비한다.

권장 환경은 다음과 같다.

- 최신 Chromium 계열 브라우저 사용
- 화면 배율 100% 권장
- 장시간 작업 시 새로고침보다 검색과 필터를 우선 사용

## 4. 로그인

### 4.1 로그인 방법

1. 브라우저에서 `https://61.109.220.75:15881/`에 접속한다.
2. 아이디와 비밀번호를 입력한다.
3. 로봇 확인 항목을 완료한다.
4. `로그인` 버튼을 클릭한다.

![로그인](manual-screenshots/login.png)

### 4.2 로그인 후 알아둘 점

- 그룹 관리자는 자기 그룹 범위로 자동 제한된다.
- 최고 관리자처럼 그룹 선택으로 다른 조직을 오가지는 않는다.
- 비밀번호 정책에 따라 변경 안내가 나타날 수 있다.

### 4.3 로그인 실패 시 점검 항목

- 아이디 오입력 여부
- 비밀번호 오입력 여부
- 계정 잠금 여부
- 비밀번호 만료 여부

## 5. 공통 화면 사용법

### 5.1 상단 헤더

상단 헤더에서는 다음 정보를 확인하거나 사용할 수 있다.

- 비밀번호 만료 예정일
- 알림 아이콘
- 프로필 메뉴

### 5.2 좌측 메뉴

그룹 관리자 기준 주요 메뉴는 다음과 같다.

- 대시보드
- 관리
- 관리자 설정
- 감사

`관리` 하위 메뉴는 다음으로 구성된다.

- 컨트롤러 관리
- 중계서버 관리
- 가상보안네트워크 관리
- 사용자 관리
- 부서 관리
- 알람 관리
- 이상감지 관리
- 요청 관리

### 5.3 그룹 관리자 화면의 중요한 차이

- 최고 관리자와 달리 `조직 관리`, `라이선스 관리`, `보안 및 정책`, `시스템품질` 메뉴가 없다.
- 그룹 선택 UI를 사용하지 않고, 현재 로그인한 그룹 범위로만 조회와 작업이 이뤄진다.
- 생성 작업 시 최고 관리자처럼 `그룹 미선택` 경고가 나타나지 않는다.

- 최근 감사 이력에서 같은 그룹 내 누가 어떤 운영 작업을 했는지 먼저 확인한다.

## 7. 관리 메뉴

## 7.1 컨트롤러 관리

### 7.1.1 진입 경로

- 좌측 메뉴 `관리 > 컨트롤러 관리`
- 직접 URL: `/system/management/controller`

### 7.1.2 이 화면에서 하는 일

- 컨트롤러 목록 조회
- 검색
- 등록
- 수정
- 삭제
- 연결 상태 확인

### 7.1.3 컨트롤러 등록 방법

1. `컨트롤러 등록` 버튼을 클릭한다.
2. `컨트롤러 이름`, `설명`을 입력한다.
3. `등록하기`를 클릭한다.
4. 목록에서 생성 결과를 확인한다.

입력 가이드

- 컨트롤러 이름: 컨트롤러 식별자 역할의 필수값
- 설명: 운영 위치, 장비 용도, 담당자 메모

![컨트롤러 등록 팝업](manual-modal-screenshots/controller-create.png)
![컨트롤러 관리](manual-screenshots/controller.png)

### 7.1.4 운영 팁

- IP와 포트를 수정하는 경우 연결 영향 범위를 먼저 확인한다.
- 변경 후에는 연결 상태와 감사 로그를 함께 확인한다.

## 7.3 가상보안네트워크 관리

가상보안네트워크 관리는 `VNET`, `엔드포인트`, `토폴로지` 3개 화면으로 구성된다.

## 7.3.1 VNET 관리

### 7.3.1.1 진입 경로

- 좌측 메뉴 `관리 > 가상보안네트워크 관리`
- 직접 URL: `/system/management/network/vnet`

### 7.3.1.2 이 화면에서 하는 일

- VNET 목록 조회
- 검색
- 생성
- 수정
- 삭제
- 접속 정책과 외부 트래픽 정책 관리

### 7.3.1.3 VNET 생성 방법

1. `VNET 생성` 버튼을 클릭한다.
2. VNET ID와 별칭을 입력한다.
3. 컨트롤러를 선택한다.
4. 필요 시 중계서버를 선택한다.
5. 암호화 방식과 CIDR을 설정한다.
6. 단말 접속 허용, 외부 트래픽 허용, 기기 인증 여부를 설정한다.
7. 삭제예정일을 입력한다.
8. 저장 후 목록에 반영되었는지 확인한다.

입력 가이드

- VNET ID: 필수 식별값
- 별칭: 운영자가 읽기 쉬운 이름
- 컨트롤러: VNET을 제어하는 장비
- 중계서버: 필요한 경우만 선택
- 암호화 알고리즘: `ARIA_256`, `AES_256`, `LEA_256`
- 암호화 모드: `CBC`, `GCM`
- CIDR 길이: `/8`, `/16`, `/24`
- 단말 접속 허용: 엔드포인트 접속 가능 여부
- 외부 트래픽 허용: 외부망 통신 가능 여부
- 기기 인증 여부: 추가 보안 장치 사용 여부
- 삭제예정일: 유예 삭제 기준 날짜

![VNET 생성 팝업](manual-modal-screenshots/vnet-create.png)
![VNET 관리](manual-screenshots/vnet.png)

### 7.3.1.4 운영 팁

- VNET ID는 운영 중 계속 사용되므로 임시 이름을 쓰지 않는다.
- 컨트롤러 선택 오류는 엔드포인트 운영에 직접 영향을 준다.

## 7.3.2 엔드포인트 관리

### 7.3.2.1 진입 경로

- 상단 탭 `엔드포인트`
- 직접 URL: `/system/management/network/endpoint`

### 7.3.2.2 이 화면에서 하는 일

- 엔드포인트 목록 조회
- 생성
- 수정
- 삭제
- 접속 허용/차단
- 외부 트래픽 허용/차단
- 사용자 할당/해제

### 7.3.2.3 엔드포인트 생성 방법

1. `엔드포인트 생성` 버튼을 클릭한다.
2. 소속 VNET을 선택한다.
3. 엔드포인트 ID와 별칭을 입력한다.
4. 역할을 선택한다.
5. TNI 주소와 정책을 설정한다.
6. 필요 시 리다이렉트 정보를 추가한다.
7. 저장 후 목록을 확인한다.

입력 가이드

- VNET: 소속 네트워크
- 엔드포인트 ID: 필수 식별값
- 엔드포인트 별칭: 표시 이름
- 역할: `HOST`, `GATEWAY`
- TNI Address: 가상 네트워크 주소
- CIDR 길이: `/8`, `/16`, `32`
- 리다이렉트 IP 추가: 게이트웨이 운영 시 사용
- 단말 접속 허용: 접속 가능 여부
- 외부 트래픽 허용: 외부망 통신 가능 여부

![엔드포인트 생성 팝업](manual-modal-screenshots/endpoint-create.png)
![엔드포인트 관리](manual-screenshots/endpoint.png)

### 7.3.2.4 운영 팁

- HOST는 실제 사용자 할당 상태를 함께 관리한다.
- GATEWAY는 리다이렉트 정보가 정확한지 반드시 확인한다.

## 7.4 사용자 관리

### 7.4.1 진입 경로

- 좌측 메뉴 `관리 > 사용자 관리`
- 직접 URL: `/system/management/user`

### 7.4.2 이 화면에서 하는 일

- 사용자 목록 조회
- 검색과 필터
- 생성
- 수정
- 삭제
- 계정 상태 확인

### 7.4.3 사용자 생성 방법

1. `사용자 생성` 버튼을 클릭한다.
2. 아이디, 이름, 이메일, 휴대폰번호를 입력한다.
3. `선택` 버튼으로 부서를 지정한다.
4. 직위를 선택한다.
5. 결재권자 여부를 설정한다.
6. 필요 시 최종결재권자를 설정한다.
7. 저장 후 목록을 확인한다.

입력 가이드

- 아이디: 로그인 ID
- 이름: 사용자 이름
- 이메일: 연락 및 식별용 주소
- 휴대폰번호: 연락처
- 부서: 소속 부서
- 직위: 인사 체계 표시값
- 결재권자로 등록: 승인 가능 여부
- 부서 관리자(최종 결재권자): 최종 승인 담당 여부

![사용자 생성 팝업](manual-modal-screenshots/user-create.png)
![사용자 관리](manual-screenshots/user.png)

### 7.4.4 운영 팁

- 사용자 삭제 전에는 할당 장치와 엔드포인트 연결 상태를 먼저 확인한다.
- 결재권자 설정은 요청 처리 흐름에 바로 반영된다.

4. 저장 후 트리에 반영되었는지 확인한다.

### 7.5.4 직위 생성 방법

1. `새 직위 생성`을 클릭한다.
2. 직위명을 입력한다.
3. `생성하기`를 클릭한다.

입력 가이드

- 부서명: 부서 표시명
- 부서 연락처: 조직 연락처
- 부서 설명: 부서 역할 설명
- 직위명: 직위 이름

![직위 생성 팝업](manual-modal-screenshots/rank-create.png)
![부서 관리](manual-screenshots/organization.png)

### 7.5.5 운영 팁

- 부서와 직위는 사용자 생성 전에 먼저 정리하는 것이 좋다.
- 구조 변경 후에는 `변경사항 저장`을 반드시 수행한다.

## 8. 관리자 설정

### 8.1 진입 경로

- 좌측 메뉴 `관리자 설정`
- 직접 URL: `/system/account`

### 8.2 이 화면에서 하는 일

- 관리자 기본 정보 수정
- 비밀번호 변경
- 그룹 데이터 동기화

그룹 관리자는 최고 관리자와 달리 `관리자 장치 목록` 영역이 보이지 않는다.

### 8.3 비밀번호 변경 방법

1. `비밀번호 변경` 버튼을 클릭한다.
2. 현재 비밀번호, 새 비밀번호, 새 비밀번호 확인을 입력한다.
3. `변경`을 클릭한다.
4. 로그아웃 후 다시 로그인한다.

입력 기준

- 영문, 숫자, 특수문자를 포함한 8~16자
- 새 비밀번호와 확인 비밀번호는 같아야 한다.

![비밀번호 변경 팝업](manual-modal-screenshots/account-change-password.png)

### 8.4 그룹 데이터 동기화

- 그룹 관리자에게는 `전체 동기화` 대신 `그룹 데이터 동기화` 버튼이 표시된다.
- 이 기능은 현재 그룹 데이터만 동기화한다.
- 대량 변경 후 화면 상태가 맞지 않거나 최신 데이터 반영이 늦을 때 사용한다.

### 8.5 운영 팁

- 비밀번호 변경 후에는 즉시 다시 로그인해 정상 접속 여부를 확인한다.
- 그룹 데이터 동기화는 반복 클릭하지 말고 결과를 확인한 뒤 다시 수행한다.

![관리자 설정](manual-screenshots/account.png)

## 10. 그룹 관리자 일일 운영 체크리스트

### 10.1 시작 점검

1. 대시보드에서 현재 그룹 사용자와 접속 현황을 확인한다.
2. 알람 관리에서 미해결 항목을 확인한다.
3. 요청 관리에서 대기 요청이 있는지 확인한다.

### 10.2 변경 작업 전 점검

1. 영향 대상 부서, 사용자, VNET, 엔드포인트를 정리한다.
2. 최근 감사 로그에서 선행 작업 이력을 확인한다.
3. 필요한 경우 승인 대상자와 협의한다.

### 10.3 변경 작업 후 점검

1. 해당 메뉴 목록에서 저장 결과를 확인한다.
2. 관련 사용자 또는 네트워크 상태를 다시 확인한다.
3. 감사 로그에서 작업 기록이 남았는지 확인한다.
4. 필요 시 그룹 데이터 동기화를 수행한다.

## 11. 자주 하는 실수와 대응

### 11.1 자주 하는 실수

- 부서와 직위를 만들기 전에 사용자를 먼저 생성하는 경우
- VNET 생성 시 잘못된 컨트롤러를 선택하는 경우
- 요청 반려 사유를 짧게 써서 재요청이 반복되는 경우
- 엔드포인트 생성 후 사용자 할당을 빠뜨리는 경우
- 변경 후 감사 로그 확인을 생략하는 경우

### 11.2 대응 원칙

- 구조 정보는 `부서 > 직위 > 사용자` 순으로 먼저 정리한다.
- 네트워크 자원은 `컨트롤러/중계서버 > VNET > 엔드포인트` 순으로 만든다.
- 변경 후에는 목록 상태와 감사 로그를 함께 확인한다.
- 문제가 의심되면 그룹 데이터 동기화를 검토한다.

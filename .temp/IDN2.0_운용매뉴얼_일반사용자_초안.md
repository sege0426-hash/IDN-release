## 1. 문서 안내

이 문서는 IDN 2.0 일반 사용자가 사용자 포털에서 필요한 기능을 스스로 사용할 수 있도록 만든 사용자 매뉴얼이다.

일반 사용자는 관리자 콘솔을 사용하지 않는다.  
일반 사용자 화면은 상단 메뉴 중심으로 구성되며, 주로 다음 업무를 수행한다.

- 내 네트워크 상태 확인
- 내 장치 상태 확인
- 네트워크 생성 또는 참여
- 멤버 확인
- 요청 확인
- 내 정보와 비밀번호 변경

문서는 실제 사용자 입장에서 다음 순서로 구성했다.

1. 어디로 들어가는지
2. 화면에서 무엇을 할 수 있는지
3. 어떤 버튼을 어떤 순서로 눌러야 하는지
4. 각 입력 항목이 무엇을 의미하는지
5. 작업 후 무엇을 확인해야 하는지

처음 사용하는 사용자는 다음 순서로 읽으면 이해가 빠르다.

1. 로그인
2. 홈
3. 네트워크
4. 요청함
5. 설정

## 2. 일반 사용자 권한

일반 사용자는 본인 계정과 본인이 참여한 네트워크 범위에서만 작업할 수 있다.

- 내 홈 화면 확인
- 참여 중인 네트워크 조회
- 내 장치 상태 조회
- 네트워크 생성 요청 또는 생성 작업
- 멤버 조회
- 받은 요청과 보낸 요청 조회
- 프로필 수정
- 비밀번호 변경
- 계정 삭제 요청

일반 사용자에게 없는 권한은 다음과 같다.

- 컨트롤러, 중계서버 직접 관리
- 조직/부서 관리
- 사용자 관리
- 보안 정책 관리
- 감사 관리
- 관리자 설정

## 3. 로그인

### 3.1 로그인 방법

1. 브라우저에서 `https://61.109.220.75:15881/`에 접속한다.
2. 아이디와 비밀번호를 입력한다.
3. 로봇 확인 항목을 완료한다.
4. `로그인` 버튼을 클릭한다.

로그인에 성공하면 일반 사용자 포털 메인 화면으로 이동한다.

![로그인](manual-screenshots/login.png)

### 3.2 로그인 후 보이는 주요 메뉴

일반 사용자 포털 상단 메뉴는 다음과 같다.

- 홈
- 네트워크
- 멤버
- 요청함
- 설정

각 메뉴의 의미는 다음과 같다.

- 홈: 최근 알림, 참여 네트워크 수, 내 장치 상태를 한 번에 확인하는 시작 화면
- 네트워크: 내가 참여한 네트워크를 조회하고 새 네트워크를 만들거나 상세 상태를 보는 화면
- 멤버: 같은 조직 또는 부서 멤버를 찾고 초대 대상을 확인하는 화면
- 요청함: 받은 요청과 보낸 요청의 승인 상태를 확인하고 처리하는 화면
- 설정: 이름, 연락처, 이메일, 비밀번호, 계정 삭제를 관리하는 화면

### 3.3 로그인 실패 시 점검 항목

- 아이디 오입력 여부
- 비밀번호 오입력 여부
- 계정 잠금 여부
- 비밀번호 만료 여부

## 4. 홈

### 4.1 진입 경로

- 상단 메뉴 `홈`
- 직접 URL: `/`

### 4.2 홈 화면에서 할 수 있는 일

- 최근 요청 및 알림 확인
- 클라이언트 다운로드
- 메뉴얼 다운로드
- 참여 중인 네트워크 수 확인
- 내 장치 상태 확인

### 4.3 홈 화면에서 확인하는 정보

- 요청 / 알림 목록
- 승인, 반려, 초대 관련 알림
- 참여 중인 네트워크 수
- 현재 장치 상태

홈은 "오늘 바로 처리할 일"을 확인하는 화면으로 이해하면 된다.  
좌측 또는 상단 영역에서는 최근 알림과 참여 네트워크 수를 보고, 네트워크 목록을 선택하면 현재 내 장치 상태를 함께 확인할 수 있다.

### 4.4 활용 방법

- 알림이 많을 때는 요청함으로 이동해 받은 요청과 보낸 요청을 정리한다.
- 장치 상태가 `오프라인`이면 네트워크 화면에서 해당 네트워크와 장치 상태를 다시 확인한다.
- 새로 초대된 네트워크가 있으면 네트워크 화면에서 참여 상태를 확인한다.

![일반 사용자 홈](manual-screenshots/user-home.png)

### 4.5 홈 화면에서 장치 상태 확인 방법

1. 홈 화면의 네트워크 목록에서 원하는 네트워크를 클릭한다.
2. 우측 상세 패널에서 해당 네트워크에 연결된 내 장치 목록을 확인한다.
3. 장치별 `온라인`, `오프라인` 상태를 확인한다.

화면에서 보는 항목

- 네트워크명: 현재 선택한 네트워크
- 장치명: 내 계정에 연결된 엔드포인트명
- 설명: 장치 설명 또는 별칭
- 상태 라벨: 현재 접속 여부

![홈 장치 상태 패널](manual-screenshots/user-home-network-detail.png)

## 5. 네트워크

### 5.1 진입 경로

- 상단 메뉴 `네트워크`
- 직접 URL: `/my-networks`

### 5.2 네트워크 화면에서 할 수 있는 일

- 내가 참여 중인 네트워크 목록 조회
- 검색과 필터 사용
- 새 네트워크 생성
- 네트워크 상세 보기
- 네트워크 수정
- 네트워크 삭제
- 엔드포인트 생성과 수정
- 멤버 초대

### 5.3 화면에서 먼저 이해할 항목

- 네트워크 이름
- 컨트롤러
- 네트워크 대역
- 암호화 알고리즘
- 기기 인증 여부
- 단말 접속 허용 여부
- 외부 트래픽 허용 여부

### 5.4 네트워크 검색과 필터

검색 기준

- 네트워크 이름
- 컨트롤러
- 네트워크 대역

필터 기준

- 암호화 알고리즘
- 기기 인증 여부

### 5.5 새 네트워크 생성 방법

1. `새 네트워크 생성` 버튼을 클릭한다.
2. 원클릭 생성 팝업에서 초대할 사용자를 선택한다.
3. 다음 단계에서 컨트롤러, 중계서버, 자동 삭제 여부를 설정한다.
4. 마지막 단계에서 생성 요약을 확인한다.
5. `생성` 버튼을 클릭한다.

실제 팝업 구조

- 1단계: 사용자 선택
- 2단계: 연결 리소스 설정
- 3단계: 생성 요약 확인

주요 입력 항목

- 초대 사용자: 네트워크에 함께 참여할 사용자
- 컨트롤러: 네트워크를 제어할 장비
- 중계서버: 필요한 경우 연결하는 중계 장비
- 자동 삭제: 삭제 예정일 자동 운영 여부
- 삭제 예정일: 네트워크 삭제 기준 날짜

![새 네트워크 생성 팝업](manual-modal-screenshots/user-network-create.png)

### 5.6 네트워크 상세 화면 여는 방법

1. 네트워크 목록에서 원하는 네트워크 카드를 클릭한다.
2. 네트워크 상세 또는 토폴로지 화면이 열린다.
3. 좌측 목록에서 장치를 선택하거나 가운데 토폴로지에서 장치를 클릭한다.
4. 선택한 장치의 상태와 제어 항목을 확인한다.

네트워크 상세 화면에서 먼저 볼 곳

- 상단 정보줄: 컨트롤러, IP 대역, 암호화 방식, 기기 인증 여부, 단말 접속 허용 여부, 외부 트래픽 허용 여부
- 좌측 장치 목록: 네트워크에 속한 엔드포인트 목록과 사용자 할당 현황
- 중앙 토폴로지: 장치 간 연결 구조와 상태
- 장치 생성 버튼: 새 장치를 추가할 때 사용
- 장치 우측 더보기 메뉴: 초대, 수정, 삭제, 할당 해제 같은 작업 실행

![일반 사용자 네트워크 상세](manual-screenshots/user-network-detail.png)

### 5.7 네트워크 상세에서 할 수 있는 작업

네트워크 상세에서는 다음 작업을 할 수 있다.

- 엔드포인트 상태 확인
- 단말 접속 허용/차단
- 외부 트래픽 허용/차단
- 엔드포인트 생성
- 엔드포인트 수정
- 멤버 초대
- 장치 할당 해제

작업 권한은 네트워크 소유자 여부에 따라 달라진다.

- 소유자: 네트워크 수정, 삭제, 장치 생성, 장치 수정, 장치 삭제, 사용자 초대, 장치 할당 해제 가능
- 장치 사용자: 본인에게 할당된 장치의 별칭 수정 가능
- 일반 참여자: 조회 위주로 사용

장치 상세 패널에서 확인하는 항목

- 장치명: 현재 선택한 엔드포인트 이름 또는 별칭
- 주소: 장치에 할당된 네트워크 주소
- 상태: 온라인 또는 오프라인
- 사용자: 현재 장치에 할당된 사용자 ID
- 역할: 장치 역할
- 단말 접속 허용: 해당 장치의 접속 허용 여부
- 외부 트래픽 허용: 외부망 트래픽 허용 여부

### 5.8 운영 팁

- 새 네트워크 생성 전에는 함께 사용할 멤버를 먼저 정리해 두는 것이 좋다.
- 네트워크 수정이나 삭제 전에는 참여 중인 멤버와 장치 상태를 먼저 확인한다.
- 장치 상태가 `오프라인`이면 네트워크 생성 문제가 아니라 실제 단말 연결 문제일 수 있다.

## 7. 요청함

### 7.1 진입 경로

- 상단 메뉴 `요청함`
- 직접 URL: `/requests`

### 7.2 요청함에서 할 수 있는 일

- 받은 요청 확인
- 보낸 요청 확인
- 상태별 필터 확인
- 대기, 승인, 반려 결과 확인

### 7.3 화면 구성

- 상단 탭: `받은 요청`, `보낸 요청`
- 상태 필터: `전체`, `대기`, `승인`, `반려`

요청 카드는 다음 정보를 중심으로 읽으면 된다.

- 배지: `결재 필요`, `검토 중`, `승인 완료`, `반려됨`
- 요청 종류: 어떤 유형의 요청인지 표시
- 자원 종류 / 자원 ID: 어떤 네트워크 또는 자원에 대한 요청인지 표시
- 요청일: 요청이 생성된 시점
- 요청자 ID 또는 결재 경로: 누가 요청했는지 또는 현재 단계가 어디인지 표시
- 하단 안내 영역: 승인 완료, 상위 결재 대기, 반려 사유 등 실제 처리 상태 표시

### 7.4 요청 확인 방법

1. `받은 요청` 또는 `보낸 요청` 탭을 선택한다.
2. 상태 필터를 눌러 원하는 상태만 본다.
3. 요청 카드에서 요청 유형, 자원 ID, 요청일, 결과를 확인한다.
4. 반려된 요청은 반려 사유를 함께 확인한다.

`받은 요청`에서 대기 상태 요청이 있는 경우에는 카드 하단에 `승인`, `반려` 버튼이 표시된다.

- 승인: 확인 팝업에서 승인 여부를 다시 묻는다.
- 반려: 반려 사유를 입력하는 팝업이 열린다.

`보낸 요청`에서는 직접 처리 버튼은 없고 현재 상태를 확인하는 용도로 사용한다.

- `검토 중`: 아직 처리되지 않은 상태
- `승인 완료`: 최종 승인된 상태
- `반려됨`: 반려 사유를 확인한 뒤 다시 요청해야 하는 상태

### 7.5 활용 방법

- 내가 만든 요청이 아직 처리되지 않았는지 확인할 때 `보낸 요청`을 본다.
- 누군가 나에게 요청한 건을 처리해야 할 때 `받은 요청`을 본다.
- 반려된 요청은 사유를 먼저 확인한 뒤 다시 요청한다.

## 8. 설정

### 8.1 진입 경로

- 상단 메뉴 `설정`
- 직접 URL: `/settings`

### 8.2 설정 화면에서 할 수 있는 일

- 프로필 정보 수정
- 비밀번호 변경
- 계정 삭제

설정 화면은 다음 탭으로 구성된다.

- 프로필
- 보안
- 계정 삭제

### 8.3 프로필 수정 방법

1. `프로필` 탭에서 이름, 이메일, 휴대폰번호를 수정한다.
2. `변경사항 저장` 버튼을 클릭한다.

입력 가이드

- 이름: 사용자 표시명
- 이메일: 연락 및 식별용 이메일
- 휴대폰번호: `010-0000-0000` 형식

### 8.4 비밀번호 변경 방법

1. `보안` 탭으로 이동한다.
2. 현재 비밀번호를 입력한다.
3. 새 비밀번호와 새 비밀번호 확인을 입력한다.
4. 저장 버튼을 클릭한다.

비밀번호 입력 기준

- 영문, 숫자, 특수문자를 포함한 8~16자
- 새 비밀번호와 새 비밀번호 확인은 같아야 한다.

현재 비밀번호가 맞지 않거나 형식 기준을 만족하지 못하면 저장되지 않는다.

![일반 사용자 설정 보안](manual-screenshots/user-settings-security.png)

### 8.5 계정 삭제 방법

1. `계정 삭제` 탭으로 이동한다.
2. 안내 문구를 확인한다.
3. `계정 삭제` 버튼을 클릭한다.
4. 확인 팝업에서 삭제 여부를 다시 확인한다.

주의사항

- 계정 삭제는 되돌릴 수 없다.
- 계정 삭제 시 관련 네트워크와 데이터에 영향이 있을 수 있다.

## 9. 일반 사용자 일일 점검 순서

1. 홈에서 최근 알림과 장치 상태를 확인한다.
2. 네트워크에서 참여 중인 네트워크 상태를 확인한다.
3. 요청함에서 대기 중인 요청이 있는지 확인한다.
4. 필요하면 설정에서 내 정보와 비밀번호를 점검한다.

## 10. 자주 하는 실수와 대응

### 10.1 자주 하는 실수

- 요청이 승인됐는지 확인하지 않고 다시 같은 요청을 보내는 경우
- 네트워크 생성 전에 초대 멤버를 정리하지 않는 경우
- 반려 사유를 확인하지 않고 같은 요청을 반복하는 경우
- 장치가 오프라인인데 네트워크 자체 문제로 오해하는 경우
- 비밀번호 형식 기준을 맞추지 못해 변경에 실패하는 경우

### 10.2 대응 원칙

- 먼저 홈과 요청함에서 현재 상태를 확인한다.
- 네트워크 작업 전에는 초대 대상과 사용 목적을 먼저 정리한다.
- 반려 요청은 사유를 보고 수정 후 다시 요청한다.
- 오프라인 장치는 네트워크 상태와 단말 상태를 함께 본다.
